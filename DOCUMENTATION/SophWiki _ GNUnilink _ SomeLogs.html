opeos's sony cdx c880r

Here are some waveforms helping me to understand some implementation bugs

The first line is the clock and the second one is the data. BusOn is not in the waveforms, because i had glitches on it. But I haven't noticed any real toggle of the BusOn signal.

The little pulses you see are byte transfers because of the snapshot scale.

  • the First wave shows a normal transmission: there is a 8ms data high window, then the master talks 3ms after le next data low. We can see that there is a 8 bit clock pulse during the data high phase. I think this corresponds the the extra 0 between packets we see in logs.

  • the second one show 2 consecutive slave breaks: there is a data low 8ms windows followed by a 8ms data high window. when the slave want to talk, it sends a datalow pulse during this 8ms data high window.

  • the third one shows a case where I think that GNUnilink did fail to frame correctly. Some slave is talking during the data high phase directly following a transmission. This was not handled by my code. This will be corrected.

What I've learnt from these waveforms, is that I must rethink about how slave break are implemented. The GNUnilink must measure the duration of the 8ms data low window so it cannot conflict with slave breaks from other slaves.

There also are some packet transmission during data high phases in the cdx-c880r. This is not the case in other HU I had.

You can download the unilink waveforms in ascii tabular format:

  • BusOn connected on 'vert'
  • Clock connected on 'rouge'
  • Data connected on 'bleu'

other signal are not connected. There may be some noise due to crosstalk.
